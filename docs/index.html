JargonSlayer Docs

JargonSlayer is a real-time English-meeting comprehension assistant for non-native speakers. It transcribes meetings, detects idioms, business slang, indirect phrasing, acronyms, proper nouns, and domain terms, then turns them into short explanation cards. After the meeting, it can generate a bilingual summary, full transcript translation, study cards, and exportable notes.

Quickstart

The fastest way to understand the product is the built-in demo. It does not need a microphone or an API key.

  1. Open the hosted app or run the project locally.
  2. Open the ≡ menu and choose 演示.
  3. Watch the transcript on the left and explanation cards on the right.
  4. Click 停止 when the demo ends.
  5. Open 纪要与导出 and generate the meeting report.

Run locally

git clone https://github.com/mianaz/jargonslayer.git
cd jargonslayer
npm install
npm run dev
# open http://localhost:3000

For a production-like local run, use npm run build and npm start.

Meeting Workflow

1. Listen

Pick a transcription engine and click 开始监听. Final transcript segments appear in the transcript panel; interim text appears in a lighter style while speech is still being recognized.

2. Read Cards

The right panel shows explanation cards for expressions and term cards for acronyms, companies, products, metrics, and technical terms. Repeated hits increment the original card instead of flooding the feed.

3. Review

Stop the meeting, generate the report, export Markdown, Anki TSV, JSON, or a Cornell note, then reopen the session later from history.

  • Click underlined expressions in the transcript to jump to their cards.
  • Select any transcript text to run an ad-hoc lookup and optionally save it to your personal glossary.
  • If speaker diarization is enabled, click speaker labels to rename them; aliases are re-applied to later updates.

cd sidecar
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

The sidecar provides live websocket transcription at ws://127.0.0.1:8765 and a local HTTP job API for recording imports. Model choices include tiny, base, small, medium, and large-v3; small is the default daily-use target.

Imports

Transcript Import

  1. Open 历史 and choose 导入文稿.
  2. Paste text or upload .txt, .srt, or .vtt.
  3. Review the parsed segments and speaker/timestamp handling.
  4. Optionally generate Chinese translation lines.
  5. Import and analyze; the result is saved as a stopped session.

Audio and Video Files

  1. Open 历史 and choose 导入录音.
  2. Use browser-local transcription or the local Whisper sidecar path.
  3. Supported recording formats include common audio files; video files have their audio extracted locally through ffmpeg.wasm before browser transcription.
  4. With the sidecar path, diarization can be used if pyannote is configured.

Video URL Import

Video URL import is local-tier only. The sidecar uses yt-dlp on your machine and runs the result through the local transcription job pipeline. It is not part of the hosted preview path.

Use URL import deliberately. Server-side fetching of third-party video raises platform-terms and legal issues, so the project keeps this path on the user's own machine rather than the hosted demo.

AI Modes and Provider Setup

Dictionary-only Mode

Turn off AI 检测 in Settings to keep transcript text local. The built-in dictionary still detects fixed expressions and terms instantly. This is the zero-key, zero-cost, fully local text path.

Hosted Preview AI

The hosted preview can use the preview server's configured model path. Text transits the project server in memory and is forwarded to OpenRouter with data_collection=allow. Treat preview AI text as shareable with the model provider.

BYOK AI

  1. Open 设置 and go to AI 检测.
  2. Choose Anthropic or an OpenAI-compatible endpoint such as DeepSeek, Qwen, OpenRouter, Poe-compatible presets, or Ollama.
  3. Set the base URL if required.
  4. Enter an API key in the UI or use a server environment variable such as ANTHROPIC_API_KEY.
  5. Choose detection and report models, test the connection, and save.
Key handling: a UI-entered key is stored in the local browser and relayed in memory through the app's API routes. It is not persisted server-side. Environment keys keep the key out of the browser entirely.

Per-task Models

Advanced settings can use different providers or models for detection, translation, and reports. The normal split is fast/cheap for live detection and stronger models for post-meeting reports.

Review and Learning

The /review page turns saved glossary and meeting data into a lightweight learning center.

  • Use the stats strip for meeting count, cumulative expressions, cumulative terms, and recent activity.
  • Use the word cloud to scan repeated expressions and terms.
  • Use the Top 10 list to focus on the most common items.
  • Use flashcards for quick practice; mark entries as known or unknown.
  • Use Anki TSV export for serious spaced repetition.
Roadmap note: v0.3.0 plans to evolve this from glossary-only mastery into a first-class learn-set and due-review loop.

Glossary and Packs

Personal Glossary

  1. Open 我的词典 from the app.
  2. Add entries manually or select transcript text and choose 加入我的词典.
  3. Saved entries participate in future detection and shadow built-in dictionary matches.
  4. Export glossary entries to Anki TSV when you want a separate review workflow.

Built-in Packs

Dictionary packs let users narrow detection by topic. Current built-in groups include core expressions, meeting flow, project execution, feedback, sales and growth, softening/criticism, academic meetings, chitchat, business terms, tech terms, and pharma/biotech.

Community Packs

Remote packs are JSON manifests installed from a GitHub raw or jsDelivr URL. They can contain expression entries and term entries; acronyms are terms with type: "acronym".

{
  "id": "biotech-terms",
  "name": "Biotech Terms",
  "version": "1.2.0",
  "terms": [{
    "term": "IND",
    "type": "acronym",
    "gloss_en": "Investigational New Drug application",
    "gloss_zh": "美国 FDA 新药临床试验申请"
  }]
}
Known gotcha: overlapping built-in and remote terms can currently double-fire. Fix cross-source term deduplication before importing large acronym packs.

Export and Backup

Session Exports

  • Markdown report, optionally with frontmatter for Obsidian/Dataview.
  • Anki TSV for study cards.
  • Full session JSON with schemaVersion: 1.
  • Copy-to-clipboard summary.
  • Cornell note export as PNG or Markdown.

Auto-export

Settings can save every session to a folder as Markdown plus JSON. This uses the File System Access API, so folder auto-save is Chromium-oriented. It is intended for local automation, Obsidian vaults, and agent workflows.

Webhook

Settings can POST { event: "meeting.saved", session } to a custom URL after each save. Receivers should return 200 quickly and process asynchronously; the app has an 8-second timeout and no retry.

Full Backup

Full backup exports sessions, glossary, and settings. Restores preview the contents before writing and sanitize risky settings such as subscription-direct connection state.

FAQ

Speech recognition is unsupported

Use Chrome or Edge for browser recognition, or switch to local Whisper.

Whisper will not connect

Confirm the sidecar terminal is running, the URL is ws://localhost:8765, and the local firewall is not blocking loopback traffic.

Detection is too noisy or too quiet

Adjust the confidence threshold, enable/disable packs, or switch AI detection on or off depending on the meeting.

Report generation is slow

Long meetings are chunked and translated in parallel; one to two minutes is normal for substantial transcripts. Cards can be exported without a report.

My project directory moved and the sidecar broke

Python virtual environments contain absolute paths. Rebuild the sidecar venv after moving or renaming the repo.

Architecture

The app is Next.js 15 with TypeScript, Tailwind, zustand, IndexedDB, and server route proxies for LLM calls. The zustand store is the shared bus for transcript segments, interim text, cards, terms, summary output, settings, and sessions.

Browser client
  transcription engines
    demo | Web Speech | local Whisper | tab audio
  zustand store
    segments | cards | terms | summaries | settings | sessions
  detection
    dictionary instant floor
    LLM scheduler and dedupe
  UI
    transcript | cards | history | summary | review

Server routes
  /api/detect
  /api/define
  /api/translate
  /api/summarize

Sidecars
  whisper_server.py
  agent_server.py
  • src/lib/types.ts owns cross-module contracts.
  • src/lib/store.ts is the app bus; STT and detection communicate through it rather than importing each other.
  • src/lib/detect/dictionary.ts is the synchronous instant floor.
  • src/lib/detect/scheduler.ts batches LLM detection and falls back to dictionary mode.
  • src/lib/detect/dedupe.ts merges repeats and upgrades dictionary hits with contextual LLM explanations.
  • src/app/api/summarize/route.ts orchestrates summary, chunked translation, and missed-item gap fill.

Sidecars

Whisper Sidecar

sidecar/whisper_server.py handles local realtime transcription over websocket and recording jobs over HTTP. It uses faster-whisper, energy-based VAD, optional audio saving, and pyannote for diarization when configured.

Speaker Diarization

Batch imports can diarize recordings. Realtime diarization is beta: the sidecar re-runs pyannote on a recent tail window, matches speakers against a stable registry, and sends label updates back to the UI without blocking transcription.

Subscription Direct

sidecar/agent_server.py is experimental and local-only. It drives the user's already-authenticated claude or codex CLI for detect/define only. Translation and summary still use the normal app routes.

  • Default loopback host: 127.0.0.1:8767.
  • Endpoints: GET /agent/health, POST /agent/detect, POST /agent/define.
  • Detect/define require loopback Origin plus the one-time connection code printed on startup.
  • Feature gate: NEXT_PUBLIC_ENABLE_SUBSCRIPTION_DIRECT=1.
  • If the sidecar is unreachable, missing login, or quota-limited, the app falls back to dictionary detection.
Environment warning: if ANTHROPIC_API_KEY is present in the agent sidecar environment, Claude calls may prefer that key over subscription login. The sidecar warns, but does not unset it.

Remote Packs

Remote packs are validated, normalized, and loaded into an in-memory registry. Settings displays them alongside built-in packs after loading.

  • Manifest requires id, name, and version.
  • Malformed entries are dropped with warnings; the whole pack only fails when the manifest itself is unusable.
  • Entry-level pack fields are ignored and overwritten with the manifest id.
  • Changing version is enough to trigger an update.
  • Users should treat remote packs as content imports from arbitrary URLs.

Large acronym sources such as a global acronym plugin should be used as candidate data, filtered by domain and quality, and published as optional packs before promoting any entries into built-ins.
