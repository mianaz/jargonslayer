# JargonSlayer 设计宪章

> **语言:** [English](../DESIGN.md) · 简体中文（英文版为权威原文，不同步时以英文版为准）

## 通用规则(适用于所有主题,包括未来皮肤)

### CJK 排印(用户凝视最久的像素)
- 中文正文:PingFang SC,**≥14px**,line-height 1.6–1.7。中文*阅读*文本(卡片、转录、摘要)绝不使用等宽或展示字体。对短 zh 界面标签(按钮、状态线、菜单项)施加 `font-mono` 是允许的:等宽字体栈不携带 CJK 字形,中文会回退到 PingFang,而 Latin/数字以等宽渲染。不要往字体栈里添加 CJK 等宽字体。
- 盘古之白:CJK 与 Latin/数字之间加半角空格("本地 Whisper",不写"本地Whisper")。
- 中文解释行是每张卡片的主角:`text-fg font-medium`,绝不降级到 `--mut` 以下。
- `--mut2`(最淡灰)只能承载装饰性元素:数字、×N 计数器、箭头、分隔符,**绝不用于中文文字**。
- 对比度:每一对文本/背景 ≥4.5:1(WCAG AA)。必须计算,不能目测。

### 交互硬标准
- 每个可交互元素都有可见键盘焦点(全局 `:focus-visible` 环)。
- 每个空态都给出具体的下一步操作(例如指向「演示」),不写纯氛围文案。
- 加载 = 与最终布局同形状的骨架屏;错误说清发生了什么和怎么修;toast 仅用于瞬态事实。
- 所有装饰性动效在 `@media (prefers-reduced-motion: reduce)` 下全部折叠,包括吉祥物在内没有例外。
- 按钮按压反馈:scale(0.97),通过 `.btn-tactile` / `.btn-terminal` 实现。

### 文案声音(前端设计规则,zh)
- 按钮文案精确描述将要发生的事:「开始监听」而不是「提交」。同一动作在整个流程中保持同一名称。
- 以用户能控制的事物来命名,不以实现来命名(「说话人分离」而不是「pyannote pipeline」)。
- 可见文案中不使用 em-dash(—)。不使用 AI 味填充词(综上所述/值得注意/首先其次最后/"不是 A 而是 B" 句式)。
- 屠龙微文案(屠龙隐喻)全应用**最多 3 处**(仅限空态/成就位)。当前:转录空态、复习空态、练习卡完成态。

### AI 味清单(每次 polish pass 逐条检查)
- 不出现紫蓝渐变辉光、不默认使用毛玻璃、不出现三等分特性卡片的套路布局。
- 只用一个图标家族(Phosphor),strokeWidth 统一;UI 中不用 emoji;不手画 icon SVG path。
- 每个主题只有一套圆角体系、一种色温;阴影色调偏向背景色相(如果该主题使用阴影的话)。

# v3 当前主题:Dark Tech · Meeting REPL(终端风)

用户决定(2026-07-06):七份探索方案中的方向 #4 作为默认主题。其余六个(青绿焦点流、水墨、羊皮纸、黑金编辑部、手绘便签、8-bit)进入可选皮肤路线图(§v3.5)。批准的视觉目标:`docs/design-explorations/preview-4-terminal.html`。

**一句话定位:** 会议是一个运行中的进程,每句话是一个输出块,每张卡片是一条 lint 诊断;黑/白/灰的机器里只有标签带颜色,像素龙是这台机器里的幽灵。

## v3.1 色彩(单一事实源;任何位置不得出现蓝调灰)

CSS 变量定义在 `globals.css` 的 `:root, [data-theme="terminal"]` 下;Tailwind 以 1:1 方式镜像(使用字面十六进制值,手动保持同步)。

| token | 值 | 角色 |
|---|---|---|
| ink | #0A0A0A | 页面画布(纯中性黑) |
| panel | #121212 | 主面板 |
| panel2 | #1A1A1A | 抬升层:对话框、弹出框 |
| panel3 | #202020 | 悬停/激活态表面 |
| edge | #262626 | 发丝线 |
| edge2 | #333333 | 加粗分割线 |
| fg | #EDEDED | 主文本 |
| mut | #9A9A9A | 次级文本(可承载中文的最低层级) |
| mut2 | #5C5C5C | 仅限装饰性微元素,绝不用于中文 |
| lab-red | #FF5F56 | 标签:俚语 slang / 危险 / 错误 |
| lab-orange | #FFAA44 | 标签:习语 idiom;**表达高亮下划线** |
| lab-yellow | #F7D51D | 标签:委婉 indirect / 警告 |
| lab-green | #4ADE80 | 标签:监听状态 / 成功 / diff 闪烁 |
| lab-purple | #C084FC | 标签:隐喻 metaphor |
| lab-cyan | #22D3EE | 标签:术语 terms;**术语高亮下划线** |
| act | #FFFFFF | 唯一的强调色:主按钮 = 白底黑字 |

分类→色彩映射(精确):idiom=orange, slang=red, phrase=green, metaphor=purple, indirect=yellow, other=中性灰;**所有 terms=cyan**(术语类型 chip 保持中性)。

规则:
1. 任何 R≠G≠B 的色相只能出现在 ≤24px 高的元素上(标签、下划线、状态点、字符符号)。大面积填充只用中性色或白色。
2. 旧版 v1 蓝色 `--acc #5B9DFF` 及所有蓝调灰已废弃。发现即替换。
3. `warn` 语义:填充使用 lab-red(仅限小元素);warn **文本**使用 `warn-soft #FF8A80`(在深色面板上满足 AA)。破坏性主按钮使用 lab-red 描边而非填充(白色是唯一允许的大面积填充强调)。
4. 迁移期别名(`acc/acc2/gold/warn`、`--font-display`)已**退役**(2026-07-06),组件中无残留引用;`warn-soft` 已提升为正式 token(规则 3)。唯一保留的 v2 遗物是 Cornell 羊皮纸导出制品,其金色/衬线值以内联方式固定:这是冻结的制品,不是迁移残留。
5. (2026-07-06) 实时转录现已通过 `.hl-term` 高亮术语卡,使用上表中预留的 lab-cyan 术语高亮下划线 token——与 `.hl-expr` 相同的点线下划线样式(圆角、`decoration-dotted`、`underline-offset-4`、悬停时低透明度背景)。全大写术语(`surface === surface.toUpperCase()`)采用大小写敏感匹配,避免与日常词汇误撞(如 IT/US/REST);混合大小写术语保持大小写不敏感。

## v3.2 字体

- **等宽是品牌身份**:JetBrains Mono(`next/font`,变量 `--font-mono-brand`,回退 SF Mono/Menlo),用于:品牌字标、英文按钮文字、数字/时间戳/计数器、英文卡片 headword、状态线、英文标签文字。
- 中文绝不使用 CJK 等宽字形:PingFang SC,正文 ≥14px,遵循通用规则。短 zh 界面标签(主按钮、状态线)使用 `font-mono` 是允许的,CJK 会回退到 PingFang(见通用规则);zh *阅读*文本保持 `font-sans`。中文微标签:12px 下限,仅限纯装饰层级。
- 全大写微标签(仅英文)+ 所有数字使用 `tabular-nums`。
- 展示衬线在本主题中已退役;`font-display` 工具类和 `--font-display` 变量已移除。Cornell 羊皮纸制品以内联方式固定 `"Songti SC", "STSong", serif`(冻结的 v2 制品)。

## v3.3 形状与结构

- 全局圆角 0–2px。微型状态点可保持圆形。
- 卡片退化为**块**:左侧 2px 状态色条 + `bg-panel` + 上下各 1px `edge` 分隔线。不使用完整边框盒、不使用圆角卡片、不使用手抄本双线装饰。
- 转录 = **Warp 风格块流**:每个片段是一个块,左侧沟槽放置等宽时间戳 + 说话人字符符号。说话人身份 = 字符符号 `$ > # % @ &` + 名称文字(从 lab 色板通过 hashSpeaker 确定性取色);不使用填充 chip。实时暂存尾部以闪烁方块光标结束(`.cursor-block`)。
- 顶栏 = **终端标题栏**:三个装饰性窗口圆点(红/橙/绿),等宽路径风格标题 `jargonslayer — {engine}·{本地|云端} — {n} cards`,右对齐 ⌘K 提示 chip。
- 底栏 = **vim 状态线**(`StatusLine.tsx`,28px,mono 12px):反色模式块 `-- LISTENING -- / -- IDLE -- / -- STOPPED --`(监听时绿底),检测模式文字,隐私声明(本地引擎显示「音频未离开本机」/ webspeech 显示「音频经浏览器厂商云端」),卡片计数器,以及吉祥物栖息位(`#mascot-perch`)在最右端。
- **汉堡菜单整合**(用户要求):顶栏只保留 [开始/停止主按钮] + 引擎药丸 + 姿态 chip + 检测模式徽标。演示/历史/复习/设置/帮助收入一个 ≡ 下拉菜单(等宽风格,图标 + 中文标签,原始 data-testid 在菜单内部保留,ESC/点击外部关闭,支持键盘导航)。

## v3.4 像素龙吉祥物 "Bit"(原创角色,可交互)

**版权红线**:必须与参考角色明确区分(原神阿蕉:黄色身体、绿色莫西干、墨镜、双足行走;Chrome 离线 T-rex:借鉴轮廓*语言*,绝不借鉴 T-rex 造型):
- 身体:炭灰 `#3A3A3A` + 磷光绿 `#4ADE80` 背鳍/腹部点缀(终端幽灵,不是黄绿色)。
- 眼睛:方形**方块光标瞳孔**,像终端光标一样闪烁,这是标志性特征。没有墨镜。
- 体型:四足、低矮细长身体(非双足);尾尖是半块字符 ▌ 像素。
- 火焰:喷出的火焰是**多色 ANSI 像素粒子**,使用五种 lab 颜色,标签体系融入角色本身。

**画质标准(用户补充 2026-07-06;参考 Chrome dino 的轮廓语言,不参考其造型):**
- 像素网格 ≥32×24(Chrome dino 约承载 44×47 的信息量;不要压缩到 ~24×16)。
- 轮廓必须一眼读出"龙":**粗颈**(接近头宽)、大头部体积、**可见的小前臂/爪子**(哪怕只有 2–3 格,但必须存在)、粗壮的腿。
- 验收标准:在 40px 实际尺寸和 4× 放大下都能读出一只胖嘟嘟的可爱龙。

**状态机**(所有动效受 reduced-motion 门控;静态清醒姿态作为回退):
- idle:尾巴摇摆 6s 循环;光标瞳孔 1.2s 脉冲;每 5–9s 随机眨眼。
- listening(`status === "listening"`):背鳍逐格亮起,像信号强度计;身体轻微起伏。
- card-burst(cards+terms 计数增加):嘴巴张开,5–8 颗多色像素粒子弧线飞向卡片面板,0.6s;快速连续时排队执行。
- sleep:30s 无事件且非监听状态 → 趴下,闭眼,等宽 `z Z` 飘在头顶;任何事件或点击唤醒。
- click:单击 = 眨眼 + 小火焰;800ms 内三击 = 彩虹大火焰 + 1s 页面边缘 ANSI 辉光(彩蛋);长按 ≥600ms = 翻肚皮,四脚乱蹬(彩蛋)。
- 实现:自包含 `PixelDragon.tsx`(SVG `shape-rendering: crispEdges`),订阅 zustand(`status`、cards+terms 长度),零业务逻辑侵入;计时器在卸载时清理;SSR 安全。

## v3.5 主题架构(皮肤路线图)

- 每个颜色/字体 token 都是 CSS 变量;组件只引用 token。`<html data-theme="terminal">` 为默认值。
- 未来皮肤 = 一组新的 `[data-theme="…"]` 变量集 + 小型主题级装饰层(横幅/水印/吉祥物换肤)。规划 id:`qinglv | shuimo | grimoire | noir | sketch | 8bit`,对应每份存档探索方案。设置页在 ≥2 个主题存在时加入主题选择器。
- 吉祥物随主题换肤(terminal=Bit;青绿=矿绿龙;水墨=墨龙;8bit=复古色板)。骨架/状态机共享;精灵图层按主题切换。
- 探索存档:七份预览 HTML + 生成的美术素材(`ink-dragon.png`、`qinglv-band.png`)存放在 `docs/design-explorations/`。

## v3.6 动效清单(封闭集合)

光标闪烁(steps)、新卡片 `.diff-flash`(绿色冲洗 → 透明,0.8s)、吉祥物状态机、菜单展开(80ms)、按钮按压 scale(0.97)。禁止:大面积呼吸渐变、视差、悬停位移 >2px。

## 沿革(已退役规范,保留作上下文)

- **v1(已退役)**:以 Warp/Linear/Superhuman 为参照校准的深色产品外壳,蓝灰色开发工具家族风。其持久贡献已吸收进上方的通用规则(CJK 排印标准、交互硬标准、文案声音、金色标注层作为唯一签名的纪律)。其色板已废弃。
- **v2(已退役)**:"泥金手抄本 × 瑞士网格"(illuminated manuscript × Swiss grid):Cinzel/宋体展示字体、双线卡片装饰、❖ 纹饰、首字下沉、纹章红、龙形水印、8px 基线网格。8px 基线纪律和 Cornell 笔记羊皮纸制品幸存;其余部分作为未来 `grimoire` 皮肤继续存在。完整规范:本文件的 git 历史(commit cfeed9d 及更早)。
- **七份探索方案(2026-07-06)**:水墨宣纸 shuimo / 羊皮纸 grimoire / 8bit / 终端 terminal / 黑金编辑部 noir / 手绘便签 sketch / 青绿焦点流 qinglü。终端风胜出作为基础;青绿探索方案的"静核心、动边缘"注意力原则(把表现力花在注意力空闲的地方:过渡、空闲、会后,绝不花在会议进行中的阅读界面上)被采纳为通用设计价值,在终端主题内部同样适用。
